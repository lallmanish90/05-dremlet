# Memory Bank Instructions

## What is the Memory Bank?

The KiloCode memory bank is a simple collection of markdown files that persist important project information across development sessions. It helps maintain context and institutional knowledge.

## How to Use the Memory Bank

### File Purposes

- **brief.md**: High-level project overview and key facts
- **product.md**: Project vision, goals, and requirements
- **context.md**: Current development status and recent changes
- **architecture.md**: Technical design and system architecture
- **tech.md**: Technology stack and setup instructions
- **tasks.md**: Repeatable workflows and common procedures

### Best Practices

1. **Keep it Updated**: Regularly update files as the project evolves
2. **Be Specific**: Include concrete details, dates, and examples
3. **Stay Current**: Focus on what's relevant now, archive old information
4. **Make it Searchable**: Use clear headings and consistent formatting

### When to Update

- After major decisions or changes
- When onboarding new team members
- Before starting new development phases
- When encountering and solving problems

### Maintenance

- Review and update monthly
- Remove outdated information
- Consolidate duplicate content
- Ensure links and references are current

## File Organization

Each file should be self-contained but can reference other files when appropriate. Use consistent markdown formatting and clear section headers.
