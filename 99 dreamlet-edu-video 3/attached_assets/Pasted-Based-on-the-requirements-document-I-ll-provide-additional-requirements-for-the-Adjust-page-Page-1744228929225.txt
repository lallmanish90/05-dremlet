Based on the requirements document, I'll provide additional requirements for the "Adjust" page (Page 01) to handle the removal of specific elements from transcript files. Here's what this feature should include:

Additional Requirements for Page 01: Adjust
6.1.3 Additional Functionality: Transcript Cleanup
6.1.3.1 Duration Information Removal
The system shall identify and remove duration information from transcript files
Pattern detection must include:
"Duration: X minutes (~Y,YYY words)"
"Duration X minutes (~Y,YYY words)" (without colon)
Any variations with different time units or word counts
Removal must be complete with no artifacts remaining
The system shall log each instance of duration information removed
6.1.3.2 Transcript Label Removal
The system shall identify and remove transcript labels from files including:
"- Transcript"
"Transcript"
Any variations with different formatting but same meaning
Removal should preserve surrounding content
The system shall log each instance of transcript label removed
6.1.3.3 Heading Removal
The system shall identify and remove headings from transcript files
Heading detection must include:
Markdown headings (lines starting with #, ##, etc.)
HTML heading tags (<h1>, <h2>, etc.)
Header-styled text (lines with subsequent underlines using = or -)
Removal must maintain document structure and flow
The system shall log each instance of heading removed
6.1.3.4 Technical Implementation
Implement regex patterns to identify these elements:
Duration pattern: Duration:?\s*\d+\s*minutes\s*\(~[\d,]+\s*words\)
Transcript label pattern: \*\*-?\s*Transcript\*\*
Heading patterns:
Markdown: ^#{1,6}\s+.*$
HTML: <h[1-6]>.*?</h[1-6]>
Underlined: ^.+\n[=\-]+$
Process files recursively through the folder structure
Maintain original file encoding (UTF-8)
Update the adjustment controls in the UI to include toggles for each type of removal
Add these operations to the batch processing workflow
Update progress indicators to reflect these additional processing steps
Expand status display to show counts of each element type removed
6.1.3.5 User Interface Updates
Add a "Transcript Cleanup" section with checkboxes for:
"Remove duration information"
"Remove transcript labels"
"Remove headings"
Include sample detection patterns with each option
Provide a preview feature to show before/after examples
Include count statistics in the processing results
6.1.3.6 Configuration Options
Allow users to enable/disable each cleanup type individually
Provide advanced options to customize regex patterns if needed
Include option to create a backup of original files before processing
6.1.3.7 Quality Assurance
Implement verification to ensure content meaning is preserved
Add options to review changes before saving
Include detailed logging of all removals for auditing purposes
